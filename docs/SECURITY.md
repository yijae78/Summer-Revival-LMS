# SECURITY.md
# FLOWING - 보안 및 개인정보보호 정책

> **버전**: v2.0 (PIPA 강화)
> **작성일**: 2026-03-13

---

## 1. 개요

이 문서는 FLOWING에서 수집하는 개인정보의 종류, 처리 방침, 보안 대책을 정의합니다. 특히 **미성년자 개인정보** 및 **건강/의료 민감정보**를 다루므로 한국 개인정보보호법을 준수해야 합니다.

---

## 2. 개인정보 수집 항목

### 2.1 수집 항목 분류

| 분류 | 항목 | 수집 대상 | 필수/선택 | 법적 근거 |
|------|------|-----------|-----------|-----------|
| **식별정보** | 이름, 생년월일, 성별 | 참가자 | 필수 | 계약 이행 |
| **연락처** | 전화번호 | 참가자/학부모 | 필수 | 계약 이행 |
| **보호자 정보** | 보호자 이름, 연락처, 비상연락처 | 미성년 참가자 | 필수 | 안전 관리 |
| **소속 정보** | 학년, 부서 | 참가자 | 필수 | 조 편성 |
| **민감정보** | 알레르기, 지병, 복용 약물, 식이제한 | 참가자 | 선택 | **별도 동의** |
| **활동 기록** | 출석, 프로그램 응답, 포인트 | 참가자 | 자동 수집 | 서비스 제공 |
| **사진/영상** | 행사 중 촬영물 | 참가자 | **별도 동의** | 초상권 |
| **인증 정보** | 구글/카카오 계정 ID | 관리자/교사 | 필수 | 인증 |

### 2.2 민감정보 특별 관리

건강/의료 정보는 **민감정보**로 분류되어 추가 보호가 필요합니다:

```
일반 정보 (이름, 연락처 등)
  → RLS: 관리자 + 담당교사 접근 가능

민감 정보 (건강, 알레르기 등)
  → RLS: 관리자만 접근 가능
  → 교사는 자기 조원의 알레르기 정보만 열람 가능
  → 참가자 본인은 자기 정보만 열람 가능
```

## 3. 개인정보보호법(PIPA) 준수

### 3.1 적용 법조항

| 법조항 | 내용 | 적용 사항 |
|--------|------|-----------|
| **제15조** (수집·이용) | 개인정보 수집 시 동의 | 참가 신청 폼에서 필수 동의 |
| **제17조** (제3자 제공) | 제3자에게 제공 시 별도 동의 | 해당 없음 (제3자 미제공) |
| **제22조 제6항** | **만 14세 미만** 아동 수집 시 법정대리인 동의 | 생년월일 기반 자동 판별 |
| **제23조** | **민감정보**(건강, 사상, 신념 등) 별도 동의 | 건강/알레르기 정보 수집 시 |
| **제28조의8~12** | **국외 이전** 시 고지·동의 | Supabase(미국) 데이터 저장 |
| **제21조** | 보유 기간 경과·목적 달성 시 파기 | 행사 종료 후 단계적 파기 |

### 3.2 미성년자 개인정보 처리 (제22조 제6항)

```
참가 신청 폼 진입
  ↓
생년월일 입력
  ↓
만 14세 미만 판별? (현재 연도 - 출생 연도 기준)
  ├── Yes → 법정대리인 동의 섹션 활성화
  │         ├── 보호자 이름
  │         ├── 보호자 연락처
  │         ├── 보호자 관계 (부/모/기타)
  │         └── ☑ 동의 체크박스 (필수)
  │              "본인은 위 아동의 법정대리인으로서
  │               개인정보 수집 및 이용에 동의합니다."
  └── No  → 본인 동의만으로 진행
```

### 3.3 필수 동의 항목 (4개 분리)

개인정보보호법에 따라 **각각 별도 동의**를 받아야 합니다. 묶음 동의는 불가합니다.

#### 동의 1: 개인정보 수집·이용 동의 (필수)

```
[개인정보 수집·이용 동의서]

수집 목적: 교회 여름행사 참가자 등록 및 운영
수집 항목: 이름, 생년월일, 성별, 학년, 연락처, 보호자 연락처
보유 기간: 행사 종료 후 1년 이내 파기

※ 동의를 거부할 권리가 있으며, 거부 시 행사 참가가 제한됩니다.

☐ 위 내용을 확인하였으며, 개인정보 수집·이용에 동의합니다. (필수)
```

#### 동의 2: 민감정보 수집·이용 동의 (선택)

```
[민감정보(건강정보) 수집·이용 동의서]

수집 목적: 참가자 안전 관리 및 응급 상황 대응
수집 항목: 알레르기, 지병, 복용 약물, 식이 제한
보유 기간: 행사 종료 후 30일 이내 파기

※ 건강정보는 「개인정보보호법」 제23조에 따른 민감정보로,
  별도의 동의가 필요합니다.
※ 동의를 거부하더라도 행사 참가에는 영향이 없으나,
  건강 관련 사고 시 적절한 대응이 어려울 수 있습니다.

☐ 위 내용을 확인하였으며, 민감정보 수집·이용에 동의합니다. (선택)
```

#### 동의 3: 사진/영상 촬영·게시 동의 (선택)

```
[초상권 및 사진/영상 게시 동의서]

촬영 목적: 행사 기록 및 교회 내부 공유
게시 범위: 본 앱 내 갤러리 (로그인 사용자만 열람 가능)
보유 기간: 행사 종료 후 1년 이내 삭제

※ 동의하지 않은 참가자는 갤러리 태그 대상에서 제외됩니다.
※ 동의를 거부하더라도 행사 참가에는 영향이 없습니다.

☐ 위 내용을 확인하였으며, 사진/영상 게시에 동의합니다. (선택)
```

#### 동의 4: 개인정보 국외 이전 동의 (필수)

```
[개인정보 국외 이전 동의서]

이전받는 자: Supabase Inc. (미국)
이전 목적: 데이터베이스 저장 및 인증 서비스 제공
이전 항목: 수집된 모든 개인정보
이전 국가: 미국
이전 방법: 네트워크를 통한 전송 (HTTPS 암호화)
보유 기간: 개인정보 수집·이용 동의 보유 기간과 동일

※ 「개인정보보호법」 제28조의8에 따라,
  개인정보가 해외(미국)에 저장됨을 고지합니다.
※ Supabase는 SOC 2 Type II 인증을 받은 서비스이며,
  AWS 인프라 위에서 운영됩니다.

☐ 위 내용을 확인하였으며, 개인정보 국외 이전에 동의합니다. (필수)
```

## 4. 데이터 보존 및 파기

### 4.1 보존 기간

| 데이터 유형 | 보존 기간 | 근거 |
|-------------|-----------|------|
| 참가자 기본 정보 | 행사 종료 후 **1년** | 다음 해 행사 참고용 |
| 건강/민감 정보 | 행사 종료 후 **즉시 삭제** (30일 이내) | 최소 보존 원칙 |
| 출석/활동 기록 | 행사 종료 후 **1년** | 통계/보고 목적 |
| 사진/영상 | 행사 종료 후 **1년** | 추억 공유 목적 |
| 인증 정보 | 계정 삭제 시 즉시 | Supabase Auth 연동 |

### 4.2 자동 파기 알림

```
행사 종료 30일 후:
  → 관리자에게 알림: "민감정보(건강/알레르기) 삭제 시점입니다."
  → [지금 삭제] [30일 연장] 버튼 제공

행사 종료 1년 후:
  → 관리자에게 알림: "참가자 데이터 보존 기간이 만료되었습니다."
  → [데이터 삭제] [아카이브] [연장] 선택
```

### 4.3 삭제 방식

```
삭제 요청 시:
  1. DB 레코드 삭제 (hard delete)
  2. Storage 파일 삭제 (사진)
  3. 삭제 로그 기록 (삭제 일시, 삭제 항목 수)
  4. 복구 불가 확인 다이얼로그
```

## 5. 데이터 접근 제어 (RLS)

### 5.1 역할별 접근 매트릭스

| 데이터 | 관리자 | 교사 | 참가자 | 학부모 |
|--------|--------|------|--------|--------|
| 이름, 학년 | 전체 | 내 조원 | 본인+조원 | 자녀 |
| 연락처 | 전체 | 내 조원 | 본인 | 자녀 |
| **건강/알레르기** | **전체** | **내 조원 (읽기만)** | **본인** | **자녀** |
| 출석 기록 | 전체 | 내 조원 | 본인 | 자녀 |
| 프로그램 응답 | 전체 | 내 조원 | 본인 | 자녀 |
| 사진 | 전체 | 업로드+열람 | 열람 | 열람 |

### 5.2 RLS 정책 핵심

```sql
-- 관리자: 전체 접근
CREATE POLICY "admin_full_access" ON participants
  FOR ALL TO authenticated
  USING (
    EXISTS (
      SELECT 1 FROM profiles
      WHERE profiles.id = auth.uid()
      AND profiles.role = 'admin'
    )
  );

-- 교사: 담당 조원만
CREATE POLICY "staff_own_group" ON participants
  FOR SELECT TO authenticated
  USING (
    id IN (
      SELECT gm.participant_id
      FROM group_members gm
      JOIN groups g ON g.id = gm.group_id
      WHERE g.leader_id = auth.uid()
    )
  );

-- 참가자: 본인만
CREATE POLICY "student_own_data" ON participants
  FOR SELECT TO authenticated
  USING (user_id = auth.uid());
```

## 6. BYOS 모델의 보안 책임

### 6.1 책임 분담

| 영역 | 책임자 | 설명 |
|------|--------|------|
| 앱 코드 보안 | 앱 개발자 | XSS, CSRF 방어, 입력 검증 |

### 6.2 사용자에게 안내할 보안 수칙

온보딩 위자드 완료 후 또는 설정 페이지에서 표시:

```
⚠️ 보안 수칙

1. Supabase anon key는 공개되어도 안전하지만,
   Service Role Key는 절대 공유하지 마세요.
2. 관리자 비밀번호를 안전하게 관리해주세요.
3. 행사 종료 후 불필요한 개인정보를 삭제해주세요.
4. Supabase 대시보드에서 정기적으로 접근 로그를 확인해주세요.
```

## 7. 사진/초상권 정책

### 7.1 동의 수집

```
참가 신청 시:
  ☑ 행사 중 촬영된 사진/영상이 교회 내부 공유 목적으로
    게시될 수 있음에 동의합니다. (선택)

  → 갤러리 업로드 시 해당 참가자 태그 금지

```
기본: 앱 내부에서만 열람 가능 (로그인 필수)

## 8. AI 챗봇 데이터 프라이버시

### 8.1 채팅 데이터 처리 원칙

```
채팅 데이터 생명주기:
  사용자 질문 → 서버 Route Handler → AI API (Gemini/Groq) → 응답 스트리밍
                                                            ↓
                                                    응답 완료 후 폐기

저장 여부:
  - DB 저장: ❌ (채팅 기록은 DB에 저장하지 않음)
  - 서버 메모리: 요청 처리 중에만 (응답 후 즉시 폐기)
  - 브라우저: 세션 메모리에만 (페이지 새로고침 시 초기화)
  - AI 제공자: Google/Groq의 자체 데이터 정책 따름
```

### 8.2 AI API 보안

| 항목 | 처리 |
|------|------|
| API 키 노출 | 서버 전용 환경변수, 브라우저 절대 노출 금지 |
| 사용자 데이터 전송 | 질문 텍스트만 전송, 개인정보(이름, 연락처) 미포함 |
| 시스템 프롬프트 | 앱 기능 안내만, 사용자 DB 데이터 미주입 |
| 프롬프트 인젝션 | 시스템 프롬프트에 역할 제한 명시 |

### 8.3 사용자 안내

```
AI 도우미 이용 시 참고사항:
- AI 답변은 참고용이며, 정확하지 않을 수 있습니다.
- 개인정보(이름, 연락처 등)를 입력하지 마세요.
- 대화 내용은 저장되지 않으며, 페이지를 닫으면 초기화됩니다.
```

## 9. 기술적 보안 대책

### 9.1 전송 보안
- HTTPS 필수 (Vercel 기본 제공)
- Supabase API 통신도 HTTPS

### 9.2 입력 검증
- **클라이언트**: Zod 스키마로 폼 검증
- **서버**: Server Action에서 Zod 재검증 (클라이언트 검증 우회 방지)

### 9.3 XSS 방어
- React의 기본 이스케이핑
- 사용자 입력 HTML 렌더링 금지 (dangerouslySetInnerHTML 사용 금지)

### 9.4 인증 보안
- Supabase Auth의 JWT (httpOnly cookie)
- 세션 만료: 1시간 (refresh token으로 자동 갱신)
- 관리자 비밀번호: bcrypt 해싱 (Supabase Auth 내장)

## 10. 개인정보 처리방침 (앱 내 게시)

앱 하단 또는 설정 페이지에 개인정보 처리방침 링크를 표시합니다.

**필수 포함 항목** (개인정보보호법 제30조):
1. 개인정보 수집 목적
2. 수집하는 개인정보 항목
3. 개인정보 보유 및 이용 기간
4. 개인정보 제3자 제공 사항 (해당 없음)
